### Task 3: Fix Build Error

**Files:**
- Fix: `STS2AIAgent/Game/GameActionService.cs` (~line 2843)

The merge will succeed at git level, but `dotnet build` will fail with:
```
CS1729: "DiscardPotionGameAction" does not contain constructor with 3 parameters
```

**Root cause:** Our local customization changed the DiscardPotionGameAction call from 3 args to 2 args (removing `CombatManager.Instance.IsInProgress`). But the merge brought back the upstream 3-arg version.

- [ ] **Step 1: Try build to confirm the error**

```bash
cd AgenticSTS/STS2-Agent-Fork
dotnet build STS2AIAgent/STS2AIAgent.csproj -c Release 2>&1 | tail -20
```
Expected: CS1729 error on DiscardPotionGameAction.

- [ ] **Step 2: Find and fix the constructor call**

Search for `DiscardPotionGameAction` in GameActionService.cs. Change the 3-arg call to 2-arg:

```csharp
// BEFORE (upstream 3-arg, broken):
new DiscardPotionGameAction(potionSlot, someArg, CombatManager.Instance.IsInProgress)

// AFTER (2-arg, matching our local fix):
new DiscardPotionGameAction(potionSlot, someArg)
```

Read the surrounding code to determine the correct 2-arg form.

- [ ] **Step 3: Build again**

```bash
dotnet build STS2AIAgent/STS2AIAgent.csproj -c Release 2>&1 | tail -10
```
Expected: Build succeeded.

- [ ] **Step 4: Commit fix**

```bash
git add STS2AIAgent/Game/GameActionService.cs
git commit -m "fix: DiscardPotionGameAction constructor 3→2 args after v0.5.3 merge"
```
