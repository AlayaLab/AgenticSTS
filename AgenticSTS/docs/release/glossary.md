# Glossary — AgenticSTS release

One-page reference for the recurring terms in the paper, code, and docs.
If you've just landed here from the public README and a term seems
opaque, start by checking it here.

## Memory architecture

| Term | Definition |
|---|---|
| **Bounded-memory contract** | The paper's core thesis: every decision prompt is freshly composed from five typed slots ($L_1$–$L_5$); raw cross-decision transcripts are never appended. Decisions see only what typed retrieval selects. |
| **Per-decision composition** | The function $u_d = \pi(L_1, L_2(s_d), L_3(s_d), L_4(s_d), L_5(s_d))$ — each LLM call receives a fresh user message rebuilt from the five layers at state $s_d$. |
| **Typed retrieval** | Each retrieved piece is labeled by its source layer (e.g., "this came from $L_5$"), enabling per-layer ablation. |
| **$L_1$ — Protocol prompts** | Fixed role/protocol templates per state type (COMBAT, DECKBUILD, STRATEGIC). Postrun-immutable since 2026-04-18 (33/33 prompt patches failed A/B). |
| **$L_2$ — State-typed schemas** | Fixed legal-action formats and per-decision-type schemas. Postrun-immutable. |
| **$L_3$ — Game knowledge** | Static enumerable game data (cards, relics, enemies, encounters, events). Refreshed per game patch via `scripts/apply_patch.py`. |
| **$L_4$ — Episodic memory** | Postrun-writable case-based recall, indexed by (character × ascension × act × enemy class). Stored in `AgenticSTS-Data/memory/`. |
| **$L_5$ — Skill library** | Postrun-writable triggered strategic guides. Mistake-driven discovery + 4-level write gate + Mode B stub-template filling. Stored in `AgenticSTS-Data/skills/`. |
| **Strategic Thread** | Per-run `StrategicNote` chain (turn/combat/run scopes) injected into every prompt as a `## Strategic Thread` block. Persists across save/quit replay. |
| **Combat truncation** | `CombatConversation.llm_messages` emits at most 3 messages per round: `[combat_start, "ok", latest_user_state]`. Prior rounds are summarized through typed state, not appended. |

## Ablation cells

The 5-cell decomposition the paper uses for the fixed-$A_0$ headline matrix
(`tab:fivecond` in the paper PDF, Table 2 in the release README):

| Cell | $L_5$ | $L_4$ | Notes |
|---|---|---|---|
| `baseline-strict` | -- | -- | Baseline prompt, strict knowledge filter, no STM, no combat conversation wrapper |
| `prompt-only` | -- | -- | Full prompts + STM + combat conv, but $L_4$/$L_5$ disabled |
| `mode-a` | hand-authored seeds | -- | Human-authored $L_5$ seed library (6 seeds for Silent) |
| `mode-b-frozen` | stub-template-filled | -- | $L_5$ bodies generated by Mode B postrun stub fill, then frozen |
| `full-frozen` | hand seeds | frozen | Mode A + frozen $L_4$ episodic store |

**Naming caveat**: `mode-b-frozen` and `full-frozen` are the *paper-headline*
names (postrun disabled, stores frozen at SHA `1888a62`). The same cells
appear under their *code-level* names (`self-evolve`, `full`) in
`scripts/run_ablation.py`, which describes the postrun-active variants.
The paper headline uses the frozen variants; the code matrix is broader.

## Mode A vs Mode B

| Mode | Source of $L_5$ skill bodies |
|---|---|
| **Mode A** | Human-authored seed JSON files committed to the repo (`src/skills/seeds/`). Six seeds for the Silent character. |
| **Mode B** | Stub-template-filled bodies generated by postrun stage 5 (`src/skills/stub_filler.py`). Five character-parametric templates (`combat`, `boss`, `deckbuilding`, `map`, `intermission`) filled with prose from the previous run + most-recent-win + most-recent-loss. |

The paper's Mode B finding: template-filled $L_5$ bodies match hand-authored
mode-a within bootstrap noise (6/10 vs 6/10 at $A_0$; score-diff CI
$[-18.6, +24.8]$). Mode B is the "agent writes its own skills from
scaffolds" operating point.

## Statistical conventions

| Term | Definition |
|---|---|
| **Wilson 95% CI** | Win-rate interval per Wilson (1927). Formula in `paper/.../sections_v2/appendix.tex` Eq. (wilson). Implemented at `scripts/reproduce/_lib.py:wilson_ci`. |
| **Bootstrap 95% CI** | Score interval via 5,000-resample percentile bootstrap per Efron (1979). Implemented at `_lib.py:bootstrap_ci`. |
| **Clopper-Pearson** | Exact binomial CI used for the descriptive pooled scaffolded row. Implemented at `_lib.py:clopper_pearson_ci`. |
| **Score formula** | $s = 100$ if victory else $\text{floor} + (52/3) \cdot \text{bosses}$. The 52/3 coefficient calibrates 3 cleared bosses ≈ 52 points (mid-Act-3 floor reach). |
| **Boss count derivation** | 0 if `floor < 18`, 1 if `floor < 34`, 2 if `floor >= 34`, 3 if `outcome == "victory"`. Implemented at `_lib.py:boss_count_from_floor`. |
| **Balanced 50-game subset** | The fixed-$A_0$ headline uses the first 10 completed games per condition by start time (5 cells × 10). Implemented at `_lib.py:filter_cell`. |
| **Streams never pooled** | Fixed-$A_0$ / cross-backbone / auto-mode-ladder / archive denominators are reported separately, never averaged together. See `paper/.../appendix.tex` § "Denominator map". |

## Difficulty + game

| Term | Definition |
|---|---|
| **Ascension $A_n$** | Slay the Spire 2's ordinal difficulty modifier; $A_0$ is the easiest tier, $A_{10}$ is the maximum. Higher Ascensions stack rule modifiers (e.g., higher enemy HP, more elites, fewer rewards). |
| **Auto-mode ladder** | Run protocol where the agent advances to $A_{n+1}$ after a win and retries $A_n$ after a loss. Endpoint metric is highest attempted ascension. |
| **Fixed-$A_0$** | Run protocol where every run targets the lowest difficulty; the metric is the reliability win-rate at one difficulty. |
| **Silent / Regent / Crowbreaker / Bombardier / Necrobinder** | The five playable characters in Slay the Spire 2. The paper's headline runs target **Silent only**, chosen to keep the typed substrate self-consistent in one submission. |

## SHAs and anchors

| Term | Definition |
|---|---|
| **SHA `1888a62`** | Sibling data repo (`AgenticSTS-Data`) commit at which all fixed-$A_0$ cells were frozen. Full SHA: `1888a62b1172073f6ce735cc22cd382e6d1eed3b`. Every paper-cited number traces back to this snapshot. |
| **Frozen stores** | $L_4$ + $L_5$ stores anchored at SHA `1888a62`. Used by every fixed-$A_0$ cell; postrun and evolution writes disabled at run time. |
